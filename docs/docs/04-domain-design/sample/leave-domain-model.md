## LeaveRequest

- 識別子: LeaveRequest
- 種類: 集約ルート
- 不変条件
  - 休暇期間は LeavePeriod の不変条件を満たす（BR-INV-001）。
  - 「申請中」の間は判断者情報（判断者社員番号・氏名・判断日時・判断コメント）は未設定である（BR-INV-002）。
  - 「承認済」または「却下済」に遷移した時点で、判断者情報がすべて設定済みである（BR-INV-002）。
  - 状態遷移は「申請中 → 承認済」または「申請中 → 却下済」のみ許可される。承認済 / 却下済からの再遷移はできない（BR-PRE-002）。
- 属性
  - id: 申請の一意識別子。
  - applicantEmpNum / applicantName / applicantOrgId / applicantOrgName: 申請時点の申請者所属情報のスナップショット。後の組織異動の影響を受けずに監査できるよう、申請受付時点の値を保全する。
  - leaveType: 休暇種別（LeaveType）。
  - period: 休暇期間（LeavePeriod）。
  - reason: 申請理由（1〜500 文字。BR-PRE-003）。
  - status: 現在の申請状態（LeaveStatus）。
  - appliedAt: 申請を受け付けた日時。
  - judgeEmpNum / judgeName / judgedAt / judgeComment: 判断時点の判断者情報のスナップショット。
- 振る舞い
  - approve(判断者, 判断日時, コメント): 現在の状態が「申請中」であることを確認した上で「承認済」へ遷移させ、判断者情報を保全する。コメントは任意。
  - reject(判断者, 判断日時, コメント): 現在の状態が「申請中」であることを確認した上で「却下済」へ遷移させ、判断者情報を保全する。コメントは必須（BR-PRE-004）。
- その他
  - ライフサイクル: create で生成され「申請中」状態となり、approve / reject のいずれか 1 回で終端状態（承認済 / 却下済）に遷移する。再申請や取消はサンプルアプリのスコープ外。
  - 同一性: id で識別する。

## LeavePeriod

- 識別子: LeavePeriod
- 種類: 値オブジェクト
- 不変条件
  - 開始日は終了日以前である（startDate ≤ endDate）。違反した状態の LeavePeriod は生成されない（BR-INV-001）。
- 属性
  - startDate: 休暇開始日（その日を含む）。
  - endDate: 休暇終了日（その日を含む）。
- 振る舞い
  - 不変条件を満たさない値の組では生成自体が拒否される。
- その他
  - 同一性: 開始日と終了日の値の組で同値判定する。

## LeaveStatus

- 識別子: LeaveStatus
- 種類: 値オブジェクト（列挙型）
- 不変条件
  - 状態は「申請中（PENDING）」「承認済（APPROVED）」「却下済（REJECTED）」のいずれかに限られる。
  - 遷移ルール: 「申請中 → 承認済」「申請中 → 却下済」のみ許可。承認済 / 却下済からの遷移は禁止（BR-PRE-002）。
- 属性
  - PENDING: 申請受付後、未判断の状態。
  - APPROVED: 判断者により承認された終端状態。
  - REJECTED: 判断者により却下された終端状態。
- 振る舞い
  - 遷移可否判定: 次に進める状態が遷移ルールに合致するかを判定する。

## LeaveType

- 識別子: LeaveType
- 種類: 値オブジェクト（列挙型）
- 不変条件
  - 種別は「有給（PAID）」「特別休暇（SPECIAL）」「代休（COMPENSATORY）」のいずれかに限られる。
- 属性
  - PAID: 有給休暇。
  - SPECIAL: 特別休暇（慶弔等）。
  - COMPENSATORY: 代休。
- 振る舞い
  - 現サンプルでは種別ごとの差異（残日数管理等）はスコープ外で、識別ラベルとして用いる。

## ApprovalPolicy

- 識別子: ApprovalPolicy
- 前提条件
  - 申請者の所属組織コードが特定済みである。
  - 判断者の所属組織コードと階層コードが取得済みである。
  - 「承認権限を持つ階層コード一覧（管理職階層コード）」がアプリケーション設定から取得済みである。
- 入力: 申請者の所属組織コード, 判断者の所属組織コード, 判断者の階層コード, 管理職階層コード一覧
- 出力: 承認可否（true / false）
- メインフロー
  1. 申請者と判断者の所属組織コードが一致することを確認する。
  2. 判断者の階層コードが管理職階層コード一覧に含まれることを確認する。
  3. 両方を満たす場合のみ true を返す（BR-POL-001）。
- 代替フロー
  a. いずれかの条件を満たさない場合
  1.  false を返す。
- 例外
  - なし（純粋関数として副作用を持たない）。
- 事後処理
  - なし。
