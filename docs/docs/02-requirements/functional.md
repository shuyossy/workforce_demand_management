# 機能要件

## 題材

社内の日常業務を模した **認証不要のタスク管理**。2 状態（TODO / DONE）の最小ワークフロー（一覧・作成・完了）を通じて、社内ライブラリ準拠 IF（`MSTBusinessException` / `MSTBusinessNonRecoverException` / `@InjectLogger`）が自然に活きる構成。本サンプルは認証機能を持たない（ログイン画面・セッション認証は存在しない）ため、全操作は匿名の単一利用者を前提とする。

## ユースケース一覧

| ID                  | 概要                               | 主アクター |
| ------------------- | ---------------------------------- | ---------- |
| ListTasksUseCase    | 全タスクを作成日時降順で一覧表示   | 利用者     |
| CreateTaskUseCase   | タイトルを入力してタスクを新規作成 | 利用者     |
| CompleteTaskUseCase | TODO のタスクを DONE へ完了させる  | 利用者     |

## ユーザストーリー

| ID    | ストーリー                                                                                             |
| ----- | ------------------------------------------------------------------------------------------------------ |
| US-01 | 利用者として、タイトルを入力してタスクを登録したい。そうすればやるべきことを記録しておける。           |
| US-02 | 利用者として、登録済みのタスクを一覧で確認したい。そうすれば未完了の作業を把握できる。                 |
| US-03 | 利用者として、完了したタスクを完了操作でマークしたい。そうすれば完了済みと未完了を区別して管理できる。 |

## 受け入れ基準

ユースケースごとに Given-When-Then 形式で記述する。文中の業務エラー文言は §業務エラーシナリオ で参照されているメッセージと一致させる。

### ListTasksUseCase

- **AC-LIST-01**: Given タスクが 1 件以上登録されている / When 一覧を取得する / Then 全タスクが作成日時降順で提示される。
- **AC-LIST-02**: Given タスクが 1 件も登録されていない / When 一覧を取得する / Then 空の一覧が提示される。

### CreateTaskUseCase

- **AC-CREATE-01**: Given タイトルが 1〜100 文字で入力されている / When 作成を確定する / Then タスクが TODO 状態で登録され、一覧画面へ遷移し、作成したタスクが一覧の先頭に表示される。
- **AC-CREATE-02**: Given タイトルが未入力または空白のみ / When 作成を確定する / Then 必須項目エラーが提示され、タスクは作成されない。
- **AC-CREATE-03**: Given タイトルが 100 文字を超える / When 作成を確定する / Then 文字数超過エラーが提示され、タスクは作成されない。

### CompleteTaskUseCase

- **AC-COMPLETE-01**: Given 対象タスクが TODO 状態 / When 完了を確定する / Then タスクが DONE 状態に遷移し、完了日時が記録され、一覧画面で完了状態として表示される。
- **AC-COMPLETE-02**: Given 対象タスクが既に DONE 状態（同一タスクへの二重完了等） / When 完了を確定する / Then 業務エラー「既に完了したタスクです」が提示され、一覧画面に留置される（状態は変更されない）。
- **AC-COMPLETE-03**: Given 指定 ID のタスクが DB に存在しない（削除機能を持たない本サンプルでの POST 偽装・データ不整合） / When 完了を確定する / Then 回復不可の業務エラーとしてエラー画面（`/error.xhtml`）へ遷移する。

## 業務エラーシナリオ

回復可能な業務エラーは `MSTBusinessException` として送出され、JSF の `FacesMessage` で現画面に提示される。回復不可能な業務エラーは `MSTBusinessNonRecoverException` として送出され、`/error.xhtml` へ遷移する。

| シナリオ ID | シナリオ                    | 発生条件                                                            | システム挙動 (画面提示メッセージ)                      | 回復可否 | 関連 BR                |
| ----------- | --------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------ | -------- | ---------------------- |
| ES-01       | 既に完了したタスクの再完了  | 完了操作時に対象タスクが既に DONE                                   | 「既に完了したタスクです」（現在の一覧画面に留置）     | 回復可   | BR-PRE-001             |
| ES-02       | 完了対象タスクが存在しない  | 完了操作時に指定 ID のタスクが DB に無い（POST 偽装・データ不整合） | `/error.xhtml` へ遷移                                  | 回復不可 | BR-PRE-002             |
| ES-03       | タイトル未入力 / 文字数超過 | 作成時にタイトルが空白または 100 文字超過                           | Bean Validation 既定メッセージ（現在の作成画面に留置） | 回復可   | BR-INV-001, BR-PRE-003 |

## 受け入れ基準とビジネスルールの対応

| 受け入れ基準   | 主たる関連 BR |
| -------------- | ------------- |
| AC-CREATE-02   | BR-INV-001    |
| AC-CREATE-03   | BR-PRE-003    |
| AC-COMPLETE-01 | BR-POL-001    |
| AC-COMPLETE-02 | BR-PRE-001    |
| AC-COMPLETE-03 | BR-PRE-002    |

## 画面構成と遷移

| 画面       | パス                | 役割                                             |
| ---------- | ------------------- | ------------------------------------------------ |
| タスク一覧 | `/tasks/list.xhtml` | 全タスクを作成日時降順で表示。行ごとに完了ボタン |
| タスク新規 | `/tasks/new.xhtml`  | タイトルを入力して作成                           |
| エラー     | `/error.xhtml`      | `<error-page>` の着地                            |

遷移：一覧 ⇄ 新規（作成後は一覧へ redirect）。完了操作は一覧画面内で完結し、完了後も一覧に留まる。

## 認証要件

本サンプルは認証機能を持たない。ログイン画面・認証 Filter・セッションベースのユーザ判定は存在せず、全画面が匿名でそのまま利用可能である（詳細は [技術設計書](../technical-design) §6 認証/認可）。

## スコープ外

本サンプルアプリは Jakarta EE ボイラープレートの最小ユースケース実証を目的とするため、以下は意図的にスコープ外とする。実プロジェクト適用時には必要に応じて追加すること。

- タスクの編集・削除
- タスクの担当者割当・優先度・期限管理
- 複数利用者の区別（ユーザごとのタスク分離）と認証・認可
- カテゴリ / タグによる分類・絞り込み
- 通知（メール / チャット）
- 監査ログの長期保全と検索 UI
- 多言語対応 / アクセシビリティ要件
