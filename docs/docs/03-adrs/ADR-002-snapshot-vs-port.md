# ADR-002 ユーザ情報のスナップショット保存（サンプル例）

:::note サンプル例
本 ADR は休暇申請サンプル（leave）固有の判断記録。適用先プロジェクトでユーザ情報のスナップショット保存が不要な場合は本ファイルごと削除する（→ [サンプル削除ガイド](../getting-started/strip-sample)）。
:::

## コンテキスト

申請一覧や詳細で **他人の名前・所属** を表示する必要があるが、本サンプルでは **ユーザマスタ DB を持たない**（dev-users.yml で代替）。社内認証サーバが前段に立つ将来構成でも、ユーザ情報の正本は外部に存在する。

申請レコードに `applicant_emp_num` だけ持っておき、表示時に `UserLookupPort` で名前を解決するか、それとも申請時に名前・所属まで保存しておくかの選択が必要。

## 決定

申請レコードに **applicant の `empNum` / `name` / `orgId` / `orgName` を snapshot 保存** する。判断時にも `judge` の同等項目を snapshot 保存する。

判定基準（`ApprovalPolicy.canApprove` の `orgId` 比較）にも **スナップショット値** を使う。

## 結果

- `UserLookupPort` 不要、シンプル
- 後の異動の影響を受けない（申請時点の所属で判定が確定する）
- 「いつ申請されたか」の事実を表示・判定とも一貫して再現できる

## 代替案

- **`UserLookupPort` 経由でリアルタイム取得**：アーキ的に綺麗だが、Port / Adapter の追加と外部 IO 経路の確保が必要で複雑度が増す。異動時の挙動も「判定基準が変わる」という意外な副作用を持つ。却下
